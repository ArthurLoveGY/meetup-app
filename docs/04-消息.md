## 1. 消息列表

**文件**：`src/pages/messages/index.tsx` (77 行)

### 功能清单

| 功能 | 状态 | 实现细节 |
|------|------|---------|
| 通知列表 | ✅ | 分页加载，每页 20 条 |
| 未读标记 | ✅ | 红点标识未读通知 |
| 标记已读 | ✅ | 点击自动调用 `markAsRead` |
| 跳转详情 | ✅ | 有 `relatedId` 跳转通知详情 |
| 加载更多 | ✅ | 上拉触发 `loadMore` |
| 加载态 | ✅ | `LoadingView` |
| 空状态 | ✅ | `EmptyState`，图标 📬 |
| 全部已读 | ❌ | Store 有 `markAllAsRead`，页面无按钮 |
| 删除通知 | ❌ | Store 有 `deleteNotification`，页面无操作 |

### 数据流

```
useLoad → useNotificationStore.fetchNotifications(true)
    → notificationService.getNotifications(1, 20)
        → GET /notifications?page=1&pageSize=20
    → notificationService.getUnreadCount()
        → GET /notifications/unread-count
    → set({ notifications, unreadCount, hasMore })

点击通知 → markAsRead(notificationId)
    → notificationService.markAsRead(id)
        → PUT /notifications/:id/read
    → 乐观更新 unreadCount -= 1
    → navigateTo notification-detail
```

### 通知类型

```typescript
type NotificationType =
  | 'friend_request'   // 好友请求
  | 'trip_invite'      // 行程邀请
  | 'trip_update'      // 行程更新
  | 'join_status'      // 参与状态变更
  | 'comment'          // 评论
  | 'reminder'         // 提醒
  | 'system'           // 系统通知
```

### 依赖关系

- **Store**：`useNotificationStore`（notifications, unreadCount, isLoading, hasMore, fetchNotifications, markAsRead, loadMore）
- **Service**：`notificationService`（getNotifications, getUnreadCount, markAsRead）
- **组件**：`EmptyState`, `LoadingView`
- **平台**：`platformService.navigateTo`

### 问题

| # | 严重度 | 问题 | 建议 |
|---|--------|------|------|
| 1 | 🟡 中 | 无全部已读按钮 | 在页面右上角添加「全部已读」 |
| 2 | 🟡 中 | 无删除通知（左滑删除） | 添加滑动删除交互 |
| 3 | 🟡 中 | 无消息分类筛选 | 添加 Tab 筛选（全部/行程/好友/系统） |

| 5 | 🟢 低 | 通知类型无图标区分 | 按类型显示不同图标 |
